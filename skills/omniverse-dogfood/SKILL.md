# Omniverse Dogfood

Use this skill to run the product as a skeptical user.

## Workflow

1. Define the user goal in plain language.
2. Start from the documented entrypoint, not an internal shortcut.
3. Follow the workflow end to end.
4. Record friction, errors, confusing states, and missing feedback.
5. Distinguish product issues from environment setup issues.
6. Recommend the smallest improvement that changes the user outcome.

## Report Shape

- `Scenario`
- `Path taken`
- `What worked`
- `Breaks or friction`
- `Evidence`
- `Fix priority`

## Rules

- Use screenshots, logs, or exact UI copy when available.
- Do not assume users know internal implementation details.
- Verify empty, loading, error, and success states when the workflow has them.
