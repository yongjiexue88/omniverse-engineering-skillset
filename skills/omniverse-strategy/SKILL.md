# Omniverse Strategy

Use this skill when the decision spans more than one task.

## Workflow

1. Define the strategic objective.
2. Identify constraints, deadlines, dependencies, and reversibility.
3. Separate immediate needs from durable platform needs.
4. Sequence work into bets, milestones, and validation gates.
5. Name risks and leading indicators.

## Output Shape

- `Objective`
- `Current position`
- `Strategic options`
- `Recommended sequence`
- `Risks`
- `Decision checkpoints`

## Rules

- Prefer strategies that preserve optionality early.
- Be explicit about tradeoffs between speed, quality, scope, and risk.
- Do not disguise an implementation plan as strategy.
